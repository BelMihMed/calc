# ТЗ: Калькулятор экономии от внедрения Bitrix GPT в Битрикс24

## 1. Общее описание

Интерактивный калькулятор, который оценивает экономический эффект от внедрения ИИ (**Bitrix GPT**) в Битрикс24. Эффект складывается из двух типов ценности:

- **Экономия на ФОТ** — высвобождение рабочего времени сотрудников, которое раньше тратилось на ручные операции.
- **Прирост выручки** — дополнительная выручка за счёт высвобожденного времени или реактивации базы (считается не во всех сценариях).

### Размещение и техническая база
- Калькулятор встраивается на **страницу сайта**.
- Вся логика расчёта выполняется **на клиенте (фронтенд), без бэкенда**.
- Контекст — функциональность ИИ Bitrix GPT внутри Битрикс24.
- **Формат поставки:** один самодостаточный **HTML-файл** (HTML + CSS + JS inline), пригодный для вставки на любую страницу (через iframe или копированием блока).
- **Визуальный стиль:** фирменный стиль **Bitrix24** — чистый светлый фон, синяя гамма, карточки сценариев.
- **CTA:** отсутствует — выводятся только расчёты и цифры, без призыва к действию и формы заявки.

### Логика взаимодействия (UX)
1. Пользователь видит список из 8 сценариев и **выбирает (чекбоксами) те, что релевантны** его бизнесу.
2. Для каждого выбранного сценария раскрываются **поля ввода** с предзаполненными значениями по умолчанию (редактируемыми).
3. По **каждому выбранному сценарию** отображается свой блок результата (экономия и/или прирост, за месяц и год).
4. Внизу — **суммарный итог по всем выбранным сценариям**: общая экономия ФОТ + общий прирост выручки, за месяц и год.
5. Общие поля между сценариями на этом этапе **не выносятся** — каждый сценарий заполняется самостоятельно.

## 2. Глоссарий и базовые формулы

Эти константы и формулы используются во всех сценариях, где считается стоимость рабочего времени.

| Параметр | Значение | Комментарий |
|---|---|---|
| Налоговая нагрузка | × 1.4 | Оклад «на руки» + 40% (НДФЛ, страховые) |
| Рабочих часов в месяце | 176 | 8 ч × 5 дней × 4 недели → факт. 8 × 22 |
| Рабочих дней в месяце | 22 | Фиксировано |
| Длина рабочего дня | 8 ч | Фиксировано |

**Полная стоимость сотрудника в месяц:**
```
ПолнаяСтоимость_мес = Оклад_на_руки × 1.4
```

**Стоимость одного часа работы:**
```
СтоимостьЧаса = (Оклад_на_руки × 1.4) / 176
```

**Стоимость одной минуты:**
```
СтоимостьМинуты = СтоимостьЧаса / 60
```

**Годовой эффект** по любому показателю:
```
Эффект_год = Эффект_мес × 12
```

> Примечание по налогам: модель «оклад на руки × 1.4» — упрощённая и принятая для маркетингового калькулятора. Она прозрачна и понятна пользователю. Строгая налоговая модель не требуется.

## 3. Сценарии

### Сценарий 1 — Расшифровка звонков и заполнение карточки сделки

**Суть:** Bitrix GPT расшифровывает звонки в CRM и автоматически заполняет поля карточки сделки. Менеджер не тратит время на прослушивание и ручное заполнение.

**Роль:** менеджер по продажам. **Тип эффекта:** экономия ФОТ + прирост выручки.

**Поля ввода:**

| Поле | По умолчанию | Редактируемо |
|---|---|---|
| Количество менеджеров | 1 | Да |
| Оклад менеджера (на руки), ₽/мес | 100 000 | Да |
| Звонков в день на менеджера | 20 | Да |
| Время обработки одного звонка, мин | 10 | Да |
| План менеджера, ₽/мес | 1 000 000 | Да |
| Коэффициент реализации прироста | 0.3 (30%) | Да |

**Формулы (на одного менеджера):**
```
СэкономленоМинут_день = Звонков_день × ВремяОбработки_мин
СэкономленоЧасов_мес  = (СэкономленоМинут_день × 22) / 60

ЭкономияФОТ_мес = СэкономленоЧасов_мес × СтоимостьЧаса

ВысвобожденоЧасов_день = СэкономленоМинут_день / 60
ДоляВысвобождения      = ВысвобожденоЧасов_день / 8
Прирост_мес            = ДоляВысвобождения × План × КоэффРеализации
```
Итог умножается на количество менеджеров.

**Эталонный пример (1 менеджер, дефолты):**
- Стоимость часа: 100 000 × 1.4 / 176 = **795,45 ₽/ч**
- Сэкономлено: 10 × 20 × 22 / 60 = **73,3 ч/мес**
- Экономия ФОТ: 73,3 × 795,45 ≈ **58 333 ₽/мес**
- Высвобождение: (10 × 20)/60 = 3,33 ч → 3,33/8 = **41,7%**
- Прирост: 41,7% × 1 000 000 × 0,3 = **125 000 ₽/мес**

### Сценарий 2 — Клиентский сервис

**Суть:** Bitrix GPT обрабатывает обращения первой линии (Открытые линии, чаты). Закрывается только часть обращений — остальные уходят оператору.

**Роль:** специалист поддержки. **Тип эффекта:** только экономия ФОТ.

**Поля ввода:**

| Поле | По умолчанию | Редактируемо |
|---|---|---|
| Количество специалистов поддержки | 1 | Да |
| Оклад специалиста (на руки), ₽/мес | 70 000 | Да |
| Обращений в день (всего) | 40 | Да |
| Из них закрывает ИИ | 30 | Да |
| Время обработки одного обращения, мин | 7 | Да |

**Формулы (на одного специалиста):**
```
СэкономленоМинут_день = ОбращенийИИ × ВремяОбработки_мин
СэкономленоЧасов_мес  = (СэкономленоМинут_день × 22) / 60
ЭкономияФОТ_мес       = СэкономленоЧасов_мес × СтоимостьЧаса
```
Прирост не считается.

**Эталонный пример (1 специалист, дефолты):**
- Стоимость часа: 70 000 × 1.4 / 176 = **556,82 ₽/ч**
- Сэкономлено: 7 × 30 × 22 / 60 = **77 ч/мес**
- Экономия ФОТ: 77 × 556,82 ≈ **42 875 ₽/мес**

### Сценарий 3 — Генерация коммерческих предложений и писем

**Суть:** Bitrix GPT пишет КП и письма клиентам вместо ручного составления менеджером.

**Роль:** менеджер по продажам. **Тип эффекта:** экономия ФОТ + прирост выручки.

**Поля ввода:**

| Поле | По умолчанию | Редактируемо |
|---|---|---|
| Количество менеджеров | 1 | Да |
| Оклад менеджера (на руки), ₽/мес | 100 000 | Да |
| Документов/писем в день | 5 | Да |
| Время на один документ, мин | 20 | Да |
| План менеджера, ₽/мес | 1 000 000 | Да |
| Коэффициент реализации прироста | 0.3 (30%) | Да |

**Формулы** — идентичны Сценарию 1 (единица работы = документ):
```
СэкономленоМинут_день = Документов_день × ВремяНаДокумент_мин
СэкономленоЧасов_мес  = (СэкономленоМинут_день × 22) / 60
ЭкономияФОТ_мес       = СэкономленоЧасов_мес × СтоимостьЧаса

ВысвобожденоЧасов_день = СэкономленоМинут_день / 60
ДоляВысвобождения      = ВысвобожденоЧасов_день / 8
Прирост_мес            = ДоляВысвобождения × План × КоэффРеализации
```

**Эталонный пример (1 менеджер, дефолты):**
- Сэкономлено: 20 × 5 × 22 / 60 = **36,7 ч/мес**
- Экономия ФОТ: 36,7 × 795,45 ≈ **29 167 ₽/мес**
- Высвобождение: (20 × 5)/60 = 1,67 ч → 1,67/8 = **20,8%**
- Прирост: 20,8% × 1 000 000 × 0,3 = **62 500 ₽/мес**

### Сценарий 4 — Оценка следования скрипту (РОП + речевая аналитика)

**Суть:** РОП обычно вручную прослушивает звонки сотрудников и оценивает соответствие скрипту, тратя ~10% рабочего времени в день. Bitrix GPT покрывает 100% звонков и делает оценку за него.

**Роль:** руководитель отдела продаж (РОП). **Тип эффекта:** только экономия ФОТ.

**Поля ввода:**

| Поле | По умолчанию | Редактируемо |
|---|---|---|
| Количество РОПов | 1 | Да |
| Оклад РОПа (на руки), ₽/мес | 200 000 | Да |
| Доля времени на оценку скриптов | 10% | Да |

**Формулы (на одного РОПа):**
```
ПолнаяСтоимость_мес = Оклад × 1.4
ЭкономияФОТ_мес     = ПолнаяСтоимость_мес × ДоляВремени
```
Прирост не считается.

**Эталонный пример (1 РОП, дефолты):**
- Полная стоимость: 200 000 × 1.4 = **280 000 ₽/мес**
- Экономия ФОТ: 280 000 × 10% = **28 000 ₽/мес**

### Сценарий 5 — Повторные продажи по базе

**Суть:** Bitrix GPT генерирует повторные продажи по существующей базе клиентов, ставит менеджерам сделки, которые те закрывают. Единственный сценарий, где эффект = **только прирост выручки** (экономия ФОТ не считается).

**Роль:** — (эффект на уровне выручки компании). **Тип эффекта:** только прирост выручки.

**Поля ввода:**

| Поле | По умолчанию | Редактируемо |
|---|---|---|
| Размер базы клиентов, шт | 1 000 | Да |
| Средний чек сделки, ₽ | 50 000 | Да |
| Конверсия в повторную продажу | 10% | Нет (фиксировано) |

**Формула:**
```
ДопВыручка_мес = РазмерБазы × Конверсия × СреднийЧек
```

**Эталонный пример (дефолты: база 1 000, чек 50 000):**
- Доп. выручка: 1 000 × 10% × 50 000 = **5 000 000 ₽/мес**

> Конверсия 10% — усреднённая, в полях не редактируется.

### Сценарий 6 — Follow-up (транскрибация общих встреч)

**Суть:** Сервис транскрибирует общие встречи и даёт три независимых блока экономии:
- **А.** Сокращает длительность встреч на 15%.
- **Б.** Сокращает количество участников на 15%.
- **В.** Снимает с ассистента работу по составлению резюме встречи — ассистент освобождается на всю длительность встречи.

**Тип эффекта:** только экономия ФОТ (три блока суммируются).

**Поля ввода:**

| Поле | По умолчанию | Редактируемо |
|---|---|---|
| Общих встреч в месяц | 8 | Да |
| Средняя длительность встречи, мин | 60 | Да |
| Среднее число участников | 6 | Да |
| Средний оклад участника, ₽/мес | 100 000 | Да |
| Оклад ассистента, ₽/мес | 70 000 | Да |

**Формулы** (блоки А и Б считаются независимо от общей базы, по 15%, не последовательно; при равном проценте численно совпадают — это ожидаемо, т.к. описывают две разные причины экономии):
```
СтоимостьМинутыУчастника = (ОкладУчастника × 1.4 / 176) / 60
СтоимостьМинутыАссистента = (ОкладАссистента × 1.4 / 176) / 60

# Блок А — короче встречи на 15%
ЭкономияА_мес = Встреч × Длительность × 0.15 × Участники × СтоимостьМинутыУчастника

# Блок Б — меньше участников на 15%
ЭкономияБ_мес = Встреч × Длительность × (Участники × 0.15) × СтоимостьМинутыУчастника

# Блок В — ассистент освобождается на всё время встречи
ЭкономияВ_мес = Встреч × Длительность × СтоимостьМинутыАссистента

ЭкономияФОТ_мес = ЭкономияА_мес + ЭкономияБ_мес + ЭкономияВ_мес
```

**Эталонный пример (дефолты):**
- Стоимость минуты участника: (100 000 × 1.4 / 176)/60 = **13,26 ₽/мин**
- Стоимость минуты ассистента: (70 000 × 1.4 / 176)/60 = **9,28 ₽/мин**
- Блок А: 8 × 60 × 0,15 × 6 × 13,26 ≈ **5 729 ₽/мес**
- Блок Б: 8 × 60 × (6 × 0,15) × 13,26 ≈ **5 729 ₽/мес**
- Блок В: 8 × 60 × 9,28 ≈ **4 453 ₽/мес**
- Итого: ≈ **15 911 ₽/мес**

### Сценарий 7 — Общее повышение эффективности (Bitrix GPT для всех)

**Суть:** Любой сотрудник использует Bitrix GPT для повышения личной эффективности — обсуждения, обратная связь, генерация идей, написание текстов и т.п. Аккуратная средняя оценка прироста эффективности — **10%**.

**Роль:** все сотрудники на портале. **Тип эффекта:** только экономия ФОТ.

**Поля ввода:**

| Поле | По умолчанию | Редактируемо |
|---|---|---|
| Всего сотрудников на портале | 1 | Да |
| Средний оклад сотрудника, ₽/мес | 100 000 | Да |
| Прирост эффективности | 10% | Да |

**Формулы:**
```
ПолнаяСтоимость_мес = СреднийОклад × 1.4
ЭкономияФОТ_мес     = ПолнаяСтоимость_мес × ПриростЭффективности × КоличествоСотрудников
```
Прирост выручки не считается.

**Эталонный пример (1 сотрудник, дефолты):**
- Экономия ФОТ: 100 000 × 1.4 × 10% = **14 000 ₽/мес** на сотрудника

### Сценарий 8 — Vibe-coding

**Суть:** Инициативные сотрудники (≈10% от общей численности) используют vibe-coding для упрощения своей работы — консервативная оценка прироста **30%**.

**Роль:** инициативные сотрудники. **Тип эффекта:** только экономия ФОТ.

**Поля ввода:**

| Поле | По умолчанию | Редактируемо |
|---|---|---|
| Всего сотрудников | 10 | Да |
| Доля инициативных | 10% | Да |
| Оклад инициативного сотрудника, ₽/мес | 100 000 | Да |
| Упрощение работы | 30% | Да (фикс. дефолт) |

**Формулы:**
```
ИнициативныхСотрудников = round(ВсегоСотрудников × 0.10)   # матем. округление до целого
ПолнаяСтоимость_мес     = Оклад × 1.4
ЭкономияФОТ_мес         = ПолнаяСтоимость_мес × Упрощение × ИнициативныхСотрудников
```
Прирост выручки не считается.

**Эталонный пример (дефолты: 10 сотрудников):**
- Инициативных: round(10 × 0,1) = **1**
- Экономия ФОТ: 100 000 × 1.4 × 30% × 1 = **42 000 ₽/мес**

**Качественный тезис (выводится текстом, без цифр):**
> После работы активных сотрудников бизнес-процессы компании в среднем ускоряются и повышают эффективность на ~20%.

## 4. Итоговый блок

После всех выбранных сценариев показывается суммарный результат:

| Показатель | Месяц | Год |
|---|---|---|
| Суммарная экономия ФОТ | Σ по сценариям | × 12 |
| Суммарный прирост выручки | Σ по сценариям 1, 3, 5 | × 12 |
| **Общий эффект Bitrix GPT** | Экономия + Прирост | × 12 |

- Экономия и прирост показываются **двумя явными частями**, затем общей цифрой.
- Под итогом — качественный тезис из Сценария 8 (если он выбран): про ускорение бизнес-процессов на ~20%.

## 5. Нефункциональные требования

- **Без бэкенда.** Все расчёты — на клиенте (JS). Никаких серверных запросов для математики.
- **Встройка** на страницу сайта (виджет/блок). Учесть возможную интеграцию в страницу Битрикс24-окружения.
- **Адаптивность** — корректное отображение на десктопе и мобильных.
- **Мгновенный пересчёт** при изменении любого поля (без кнопки «Рассчитать» или с ней — на усмотрение, но дефолт — реактивный пересчёт).
- **Все денежные значения** — в рублях, с разделением разрядов и округлением до целых рублей в выводе.
- **Дефолты предзаполнены**, поля валидируются (неотрицательные числа; «закрывает ИИ» ≤ «обращений всего» в Сценарии 2).
- **Название ИИ** во всём интерфейсе — **Bitrix GPT**.

## 6. Решения по уточнённым вопросам / допущения

**Согласовано:**
- **Формат поставки** — один самодостаточный HTML-файл (HTML+CSS+JS inline).
- **Стиль** — фирменный Bitrix24 (светлый фон, синяя гамма, карточки).
- **CTA** — не добавляем; только расчёты и цифры.
- **Дефолты для С5 и С8** — заполнены демо-значениями, чтобы при выборе сценария сразу выводились ненулевые цифры: С5 — база 1 000, чек 50 000 ₽; С8 — всего сотрудников 10 (→ 1 инициативный).

**Допущения:**
- Сценарий 8: при дробном числе инициативных сотрудников применяется математическое округление до целого.
- Сценарий 6: блоки А и Б при равном проценте (15%) численно совпадают — это ожидаемо, они описывают две разные причины экономии.
- Налоговая модель — упрощённая (× 1.4), согласована как достаточная для маркетингового калькулятора.
- Коэффициент реализации прироста (0.3) и конверсия повторных продаж (0.1) — настраиваемость уточнена в полях каждого сценария.
